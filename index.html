Day 01 Vibe Coding 从听到用 查看笔记
主题一句话
从真实痛点出发，用最小产品验证价值。
用到的工具
ChatGPT、Claude Code、秒哒-笔记助手、Cursor、GitHub、Vercel、Supabase、Google Analytics、Google Search Console、百度站长工具、阿里百炼、Stripe、Enter。
最重要的 1 条方法论
凡产品皆「用户」，凡工程皆「产品」。
我接下来要怎么做
先从一个真实痛点出发，动手做一个最小可用版本来验证需求。
Day 02 用 Enter 把点子做成第一个 MVP 查看笔记
主题一句话
小步快跑，先有一版。
用到的工具
Enter、Supabase、v0、Bolt、Lovable、Google Analytics（GA）。
最重要的 1 条方法论
先验证用户需求，再搭建最小可行产品（MVP）。
我接下来要怎么做
用今日学到的工具，立即发布一个可访问的 MVP 来获取真实数据。
Day 03 会调 API 的网页：从页面到简单 Agent 查看笔记
主题一句话
让 AI 成为产品中一个可调用、可集成的模块。
用到的工具
阿里百炼、cURL、智能体 API、Supabase 函数调用、飞书通知流（Webhook）。
最重要的 1 条方法论
将高频、稳定的工作流固化为 Skills 或 API 调用，让 AI 成为可复用的产品能力模块。
我接下来要怎么做
选择一个你手动、重复的操作，尝试用今日学到的 API 或智能体服务将其自动化。
Day 04 AI 编程从 Chat 到 Agent 查看笔记
主题一句话
掌握与 AI 智能体协作编程的新范式，提升解决问题的效率。
用到的工具
ChatAI、Agent、MCP、Skills、Cursor、Plan 模式、闪电说。
最重要的 1 条方法论
根据任务场景（模型智商、操作权限、自主性）选择合适的 AI 编程工具，而非追求单一最好的工具。
我接下来要怎么做
立即使用 Cursor、Claude 等 AI 编程工具，尝试开发一个解决身边痛点的小工具或脚本。
Day 05 产品开发流程初识 查看笔记
主题一句话
与 AI 协作者一起，规划产品从 0.1 到 1.0 的生长路径。
用到的工具
产品开发流程设计图、版本规划 Prompt、写 PRD 的 Skill、ASCII 图、色值（配色）。
最重要的 1 条方法论
与 AI 协作规划产品版本，遵循先做最小可用版本（V0.1）验证核心价值，再逐步迭代的原则。
我接下来要怎么做
使用课程提供的版本规划助手 Skills，为你想做的一个产品梳理出从 V0.1 到 V1.0 的清晰路线图。
Day 06 上线与部署实战：GitHub + Vercel 踩坑记录 查看笔记
主题一句话
跨越最后一步，让作品能被真实世界访问。
用到的工具
GitHub、Vercel、Railway、Google AI Studio、Pinterest。
最重要的 1 条方法论
部署遇到问题，遵循不懂就问 AI，观察日志，用事实缩小范围的排查路径，动手解决而非空想。
我接下来要怎么做
使用 GitHub + Vercel 等工具链，将你的项目部署到公网，获得一个可访问的链接。
Day 07 MVP 极简与收敛：从能用到能讲清楚 查看笔记
主题一句话
告别玩具心态，打造一款能陪你（和用户）一起长大的真实产品。
用到的工具
方法论（需求挖掘 / MVP 设计 / 冷启动 / 迭代优化 / 增长策略 / 踩坑复盘）、ChatGPT、Skill、PowerShell、ffmpeg。
最重要的 1 条方法论
从自身真实痛点出发设计 MVP，并通过用户反馈 - 数据驱动 - 迭代优化的循环让产品持续生长。
我接下来要怎么做
在你已有的项目或新想法中，选定一个值得长期打磨的场景，开始你的第一次正式迭代。

MVP 成果展示

只展示：痛点/场景、MVP 范围、下一步迭代。

MVP 01

Prompt 记录器 / Prompt 库

痛点 / 场景
帮助用户把已经验证过的好想法安静保存下来、不打断当下专注，并在真正需要时由用户亲自唤醒的认知工具。
MVP 范围
  • 新建与保存提示词
  • 基础分类管理
  • 关键词搜索查找
下一步迭代
增加 LLM 的 API 接入，扩展更多智能化功能。
MVP 02

简历 / 岗位匹配生成器

痛点 / 场景
帮助多年未求职的职场人快速生成符合当前招聘市场标准的专业简历，解决简历写法过时和重复填写多个平台的痛点。
MVP 范围
  • 收集关键信息的表单
  • 一键生成简历的触发按钮
  • 输出结构化简历文本
下一步迭代
通过对话式 Q&A 自动补全与填写信息，帮用户更轻松生成一份更专业的简历。
